# Source Quality Tiers

## Tier 1 (высший приоритет)

Использовать для Executive Summary и ключевых выводов.

- Официальные отчёты компаний (annual reports, press releases)
- SEC filings
- Crunchbase verified data
- Peer-reviewed исследования
- Официальная документация

## Tier 2

Использовать для body отчёта.

- Аналитические отчёты (a16z, Bessemer, Sequoia, McKinsey)
- Интервью с founders (подкасты, статьи)
- Verified news (TechCrunch, Bloomberg, WSJ)
- Industry reports (Gartner, Forrester)

## Tier 3 (с осторожностью)

Использовать только для общих цифр, с пометкой источника.

- Блог-посты
- Агрегаторы (market.us, technavio, statista)
- Wikipedia
- Социальные сети
- Непроверенные новости

---

## Правила использования

1. **Executive Summary** — только Tier 1-2
2. **Key findings** — Tier 1-2, можно Tier 3 с пометкой
3. **Body** — любой Tier с указанием
4. **Recommendations** — основаны на Tier 1-2

---

## Формат источников

### ❌ НЕ ПРИНИМАЕТСЯ
```
market.us
technavio
"по данным исследований"
```

### ✅ ПРИНИМАЕТСЯ
```
Vertical AI Market Report 2024 - https://market.us/report/vertical-ai-market/ - Dec 2024
Harvey AI Raises $80M - https://techcrunch.com/2024/... - Jan 2024
SEC Filing - https://sec.gov/... - Q4 2024
```

---

## Проверка источника

Перед включением факта:

1. URL работает?
2. Дата публикации есть?
3. Автор/издание известны?
4. Можно ли верифицировать независимо?
