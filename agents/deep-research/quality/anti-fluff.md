# Anti-Fluff Rules

Правила для удаления воды и generic контента.

---

## "So What?" Test

**Применять к КАЖДОМУ факту перед включением.**

### Вопрос: "И что с этого?"

Если факт не ведёт к:
- Конкретному действию
- Принятию решения
- Выбору между опциями

→ **НЕ ВКЛЮЧАТЬ**

### Примеры

❌ **Плохо:**
```
Рынок vertical AI = $12.9B
```
So what? Как это помогает?

✅ **Хорошо:**
```
Рынок vertical AI = $12.9B, но 60% = healthcare + legal где уже гиганты.
Оставшиеся 40% (~$5B) распределены между 15+ вертикалями = возможность.
```

---

## Concrete Examples Rule

**Каждое утверждение ДОЛЖНО иметь пример.**

❌ **Плохо:**
```
Point solution быстрее выходит на рынок
```

✅ **Хорошо:**
```
Point solution быстрее выходит на рынок.
Пример: EvenUp начал с одной фичи (demand letters) и за 2 года достиг $50M ARR.
```

**Если нет примера → пометить как hypothesis, не факт.**

---

## Phrases to Remove/Rewrite

| Фраза | Что делать |
|-------|-----------|
| "важно отметить что..." | Удалить, написать суть |
| "следует рассмотреть..." | Что конкретно? |
| "рекомендуется изучить..." | Изучить ЧТО и ГДЕ (URL) |
| "необходимо провести анализ..." | Какой анализ? Первый шаг? |
| "существует возможность..." | Какая? Примеры? |
| "можно предположить что..." | Hypothesis, нужен факт |
| "в целом..." | Удалить |
| "как правило..." | Конкретика или удалить |

---

## Checklist перед публикацией

- [ ] Нет фраз из таблицы выше
- [ ] Каждая рекомендация имеет первый шаг
- [ ] Все числа имеют источник и год
- [ ] Списки не содержат очевидных пунктов (>50%)
- [ ] Каждое утверждение можно проверить

---

## Golden Rule

> Если предложение можно удалить без потери смысла — удалить.
