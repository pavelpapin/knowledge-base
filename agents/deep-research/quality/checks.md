# Quality Checks

Чеклисты для каждого стейджа. **Не переходить к следующему стейджу без прохождения чеклиста.**

---

## После Stage 2 (Data Collection)

- [ ] Каждый факт имеет прямой URL (не название сайта)
- [ ] Числа имеют год/дату (данные 2023 ≠ 2025)
- [ ] Для каждой сущности (компания, рынок) минимум 3 data points
- [ ] Source tier указан для каждого факта
- [ ] Нет "мёртвых" ссылок

---

## После Stage 3 (Fact Checking)

- [ ] ≥70% фактов verified (2+ источника)
- [ ] Все rejected facts удалены
- [ ] Unverified факты помечены
- [ ] Нет противоречий между verified фактами

---

## После Stage 4 (Synthesis)

- [ ] Каждая рекомендация отвечает на "что делать ЗАВТРА"
- [ ] Нет generic фраз без конкретики
- [ ] Framework привязан к контексту пользователя из Discovery
- [ ] Каждый вывод подкреплён примером
- [ ] "So What?" test пройден для всех фактов

---

## После Stage 5 (Devil's Advocate)

- [ ] ≥3 риска документировано
- [ ] ≥1 failed example найден
- [ ] Для каждой рекомендации есть контраргумент
- [ ] Blind spots перечислены
- [ ] Obsolescence risk оценён

---

## После Stage 6 (Final Report)

- [ ] Можно ли ДЕЙСТВОВАТЬ по отчёту без доп. ресёрча?
- [ ] Есть конкретные URLs, имена, контакты?
- [ ] Убрано всё что не помогает принять решение?
- [ ] Executive Summary только из verified facts?
- [ ] Anti-Fluff checklist пройден?
- [ ] Все источники в правильном формате?

---

## После Stage 7 (Review)

- [ ] Score ≥ 80?
- [ ] Self-Verification Questions все "да"?
- [ ] Notion page accessible?
- [ ] Пользователь уведомлён?
